# Contributing to MCPF Specification

Thank you for your interest in contributing to the MCP Trust Framework!

## Ways to Contribute

- **Report Issues:** Found a bug or inconsistency? [Open an issue](https://github.com/MCPTrustFramework/MCPF-specification/issues)
- **Suggest Improvements:** Have ideas for the specification? Start a [discussion](https://github.com/MCPTrustFramework/MCPF-specification/discussions)
- **Submit Pull Requests:** Code or documentation improvements welcome!

## Contribution Process

1. **Fork** the repository
2. **Create a branch** (`git checkout -b feature/improvement-name`)
3. **Make changes** and commit (`git commit -am 'Add improvement'`)
4. **Push** to your fork (`git push origin feature/improvement-name`)
5. **Open a Pull Request**

## Specification Changes

For changes to the specification documents:

1. Ensure technical accuracy
2. Maintain W3C/IETF standards compliance
3. Update cross-references if needed
4. Add examples where helpful
5. Update diagrams if architecture changes

## JSON Schema Changes

For schema modifications:

1. Validate JSON syntax
2. Test with example files
3. Ensure backward compatibility (or document breaking changes)
4. Update documentation

## Code of Conduct

Be respectful, constructive, and collaborative. We're building open standards for everyone.
